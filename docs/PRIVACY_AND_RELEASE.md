# 隱私、無障礙揭露與發布評估

## 目前資料行為

- App 未宣告網路權限。
- 延遲秒數與圓點座標只存於 App 私有的 `SharedPreferences`。
- 程式碼未收集、上傳或分享個人資料，也沒有分析或廣告 SDK。
- 無障礙服務不讀取畫面節點或文字，只以 `dispatchGesture` 在使用者指定座標送出點擊。

此說明必須在每次加入 SDK、網路功能、記錄或資料同步後重新審查。

## 建議的使用者揭露

在啟用無障礙服務之前清楚顯示：

> PikClick 使用無障礙服務，僅在你點選懸浮圓點後，於圓點位置執行兩次點擊。App 不會讀取畫面內容，也不會在未經你操作時自動開始。

另應提供可公開存取的隱私政策頁面，說明資料不離開裝置、停用方式、聯絡管道與政策日期。README 不是正式商店隱私政策頁面的替代品。

## 發布通路評估

### 私人／直接 APK 分發

適合封閉測試，速度最快。需提供 APK、SHA-256、版本說明及安裝警告，並保護簽章金鑰。使用者必須承擔未知來源安裝流程。

### GitHub Releases

適合公開測試與版本留存。建議發布簽章 APK、`.sha256`、變更紀錄與已知限制；正式簽章材料不可放入 CI 儲存庫。

### Google Play

觸及範圍最大，但無障礙 API 屬高審查風險。上架前必須依當時 Google Play 政策確認用途是否符合資格，完成無障礙用途聲明、顯著揭露、同意流程、Data safety 與隱私政策。政策會變動，送審時需重新查核官方規範。

- [ ] 將正式 keystore 存放在加密且有備份的位置
