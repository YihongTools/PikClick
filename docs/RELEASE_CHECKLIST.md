## 核心流程

- [ ] 首次啟動可引導取得懸浮窗權限
- [ ] 首次啟動可引導啟用無障礙服務
- [ ] 3.0、3.5、10.0 秒皆可保存並正確倒數
- [ ] 小於 3、大於 10、空白及非數字輸入會被拒絕
- [ ] 圓點可拖曳且重新開啟後位置保留
- [ ] 第一次與第二次點擊位置正確
- [ ] 倒數中再次點圓點只會取消，不產生殘留點擊

## P1 生命週期與異常測試

### 旋轉螢幕

- [ ] 直向顯示圓點後旋轉橫向，圓點仍在可視範圍
- [ ] 橫向拖曳並保存後旋回直向，圓點不會永久落在畫面外
- [ ] 倒數期間旋轉，確認只產生預期次數的點擊；若系統取消流程，畫面狀態可恢復

### 服務被系統終止

- [ ] 倒數期間以開發者選項／ADB 終止程序，不會在重啟後補送過期點擊
- [ ] 系統重啟 Sticky service 時，沒有權限便立即停止
- [ ] 重開 App 後可重新顯示圓點並完成流程

### 權限中途撤銷

- [ ] 顯示圓點後撤銷懸浮窗權限，服務不崩潰且重新進入 App 顯示未授權
- [ ] 第一次點擊後關閉無障礙服務，第二次點擊失敗並回復「開始」狀態
- [ ] 重新授權後不必清除 App 資料即可恢復

### 連續操作

- [ ] 快速連點圓點不會排入多組倒數
- [ ] 開始、取消、立即再開始，舊 callback 不會影響新流程
- [ ] 連續完成 20 組流程，沒有重複點擊、無回應或明顯記憶體增長
