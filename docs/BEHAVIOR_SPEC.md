# PikClick 行為規格

## 功能目標

由使用者主動開始一組兩次點擊。每次點擊都使用執行當下的懸浮圓點中心；倒數期間允許移動或旋轉以選擇第二次的新位置。任何必要權限撤銷或服務終止都必須取消未完成序列。

## 使用情境

- 使用者授權懸浮窗與無障礙服務，設定 0.1–60.0 秒後顯示圓點。
- 點選圓點後立即在當下中心執行第一次點擊並開始倒數。
- 使用者可保持位置、拖曳到新位置或旋轉螢幕。
- 倒數結束時重新取得圓點中心並執行第二次點擊。
- 倒數期間再次點選、撤銷權限、Accessibility service 終止或 Overlay service 終止時取消。

## 功能與非功能需求

1. 延遲必須介於 0.1–60.0 秒，預設 3.5 秒，顯示至小數第一位。
2. 每次序列只能有一個有效 token；舊 callback 不得改變新序列。
3. 第二次點擊不得快取第一次座標，必須重新讀取可視圓點中心。
4. 旋轉後圓點必須限制在新顯示範圍內，並保存新座標。
5. 缺少懸浮窗或 Accessibility service 時不得送出手勢。
6. 無障礙服務不得讀取視窗內容，只使用 gesture dispatch。
7. 啟用 Accessibility 設定前必須顯示用途、資料行為與新位置語意。
8. 使用者偏好的絕對座標不得透過備份或裝置轉移還原。
9. UI 依系統語系載入繁體中文或英文，不提供 App 內獨立語系覆寫。

## 邊界條件

- 無效、NaN、無限或超界秒數不得保存。
- 圓點尺寸尚未量測、中心超出顯示範圍時不得點擊。
- 拖曳、旋轉與倒數 callback 接近同時發生時，以 callback 執行當下可取得的中心為準。
- 權限在 dispatch 前撤銷時取消；手勢 dispatch 後才撤銷則以 Android 回呼結果為準。
- 程序被 force-stop 後不得恢復或補送舊序列。

## 驗收標準

- Given 圓點位於 A，When 第一次點擊後移到 B 且倒數完成，Then 第二次點擊使用 B。
- Given 倒數進行中，When 螢幕旋轉且圓點被限制到 C，Then 第二次點擊使用 C。
- Given 第一次點擊已完成，When 任一必要權限撤銷，Then 第二次點擊不送出且 UI 回到開始狀態。
- Given 舊序列 callback 延遲回傳，When 新序列已開始，Then 舊 callback 不改變新序列。
- Given 系統 Locale 為 `zh-Hant-TW` 或 `en-US`，When App 載入，Then 顯示對應資源字串。

## 風險與降低方式

| 風險 | 影響 | 可能性 | 降低方式 |
|---|---|---|---|
| 第二次新位置造成誤觸 | 高 | 中 | 顯著揭露、倒數文字、README 與安全警告一致 |
| 撤權與 callback 競態 | 高 | 低 | 每次 dispatch 前重驗權限、序列 token、onDestroy 取消 |
| OEM 懸浮服務差異 | 中 | 中 | API 26/30/35 emulator 加 Android 8/11/13/15 實機矩陣 |
| 絕對座標跨裝置失效 | 中 | 低 | backup/data extraction rules 排除偏好設定 |
